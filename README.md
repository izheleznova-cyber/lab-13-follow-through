#  Лабораторная работа №13: Follow Through (Финальная)

> **Девиз семестра:** *«Если замахнулся для удара — не останавливайся. Доведи до конца.»*

## 🎯 Цель работы
Сформировать навык **Follow Through** (доведения технической задачи до конца по регламенту). Освоить контейнеризацию, версионирование и прослеживаемость результатов — ключевые компетенции современного оператора и инженера БПЛА.

---

##  Почему такие требования? (Профессиональный контекст)
В авиации и UAV-индустрии **процесс важнее единичного результата**. Требования этой работы моделируют реальные стандарты:

| Требование в лабе | Аналог в индустрии БПЛА |
|------------------|------------------------|
| Путь `student-XXX/группа/lab-13-<name>/` | **Workstation ID + Operator ID** (прослеживаемость рабочей станции) |
| `git commit` после каждого этапа | **Flight Log / Mission Plan Version** (версионирование миссии) |
| Запуск через Docker | **Изолированная среда планирования** (гарантия воспроизводимости) |
| `execution_manifest.json` | **Pre-flight Check Signature** (цифровая подпись контейнера) |
| Скриншот проводника с полным путём | **Chain of Custody** (юридическая фиксация выполнения) |

📖 *Соответствует принципам ISO 21384-3 (UAV operations), ICAO Doc 10019 и современным GitOps-практикам в аэрокосмической отрасли.*

## 🛠 Подготовка
1. Убедитесь, что установлены: `Python 3.10+`, `Docker Desktop/Engine`, `Git`

2. Создайте свой репозиторий из шаблона или клонируйте выданный
3. Запустите генератор индивидуальных заданий:
   ```bash
   python3 task_generator.py

   Скрипт запросит ваше имя и создаст 9 персональных задач (3 уровня сложности). 

   Дальше вы знаете что делать)) 

   📋 Ход работы
🟢 Занятие 1: Задачи 1–6 (Модификация кода)
Откройте game.py и файлы в папке tasks/
Внесите правки только в существующие файлы. Создание новых .py файлов = нарушение регламента
После каждой решённой задачи делайте коммит:
git add tasks/task1.py && git commit -m "Task 1: velocity check [PASS]"

Занятие 2: Задачи 7–9 (Интеграция элементов)
Добавьте требуемые функции/классы в структуру проекта
Протестируйте локально: python3 game.py
Убедитесь, что код не ломает основную логику

🐳 Финальный запуск
Задайте переменную: export STUDENT_ID=<ваше_имя_латиницей>
Соберите и запустите контейнер:
docker-compose up --build 

Контейнер автоматически создаст:
traject_<name>.png (график миссии)
execution_manifest.json (цифровая подпись среды)
Убедитесь, что файлы появились в проброшенной папке ./output/ на вашем ПК

✅ Чек-лист сдачи (Follow Through Standard)
Работа принимается только при полном совпадении со списком:
Все 9 задач выполнены в коде (проверяется запуском)
В репозитории нет новых .py файлов, только правки исходных
execution_manifest.json содержит "container": "lab-13-<name>" (не "local")
График traject_<name>.png лежит в папке результатов
Скриншот проводника с чётко видимым полным путём:

Ubuntu-22.04 / home / student-XXX / группа-XX / lab-13-<name> / output /

* История Git содержит минимум 6 осмысленных коммитов с метками задач
* Файлы закоммичены и отправлены в репозиторий (git push)
⚠️ Без видимого пути на скриншоте работа не принимается.

🆘 Помощь и типичные ошибки
Ошибка
Решение
ModuleNotFoundError
Проверьте requirements.txt и docker-compose.yml volumes
Файлы не появляются в ./output/
Убедитесь, что в docker-compose.yml прописан volumes: - ./:/app/workspace
Git не принимает коммит
Настройте git config user.email и user.name
Скриншот не проходит верификацию
Включите отображение пути в проводнике: View → Show → Path bar
